# Contributing to mrtparser

Thank you for your interest in contributing to `mrtparser`! To maintain code quality and stability, please follow these guidelines:

## Contribution Process

1. **Write Tests**: Any new features or bug fixes must include corresponding tests.
2. **Verify Tests**: Ensure that all existing and new tests pass before submitting your changes. You can run tests using:

   ```bash
   make -C build test
   ```

3. **Submit a Pull Request**: Once your code is ready and tests are passing, please send a Pull Request (PR) to the project owners.

## Code Style

Please ensure your code follows the established style in the repository (supported by `.clang-format`).

We appreciate your help in making `mrtparser` better!
